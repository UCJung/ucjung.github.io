# 2026-03-29 개발일지

## 1. Heading(H1~H6) 폰트 웨이트 적용 안 됨
- **현상**: `_typography.scss`에 H1~H6 font-weight를 계층적으로 설정했으나 화면에 반영되지 않음. 개발자 도구로 확인하니 모든 heading이 `font-weight: 300`으로 표시됨
- **문제**: MDB(Material Design Bootstrap)의 CSS가 `h1~h6 { font-weight: 300 }` 으로 강제 덮어쓰고 있었음
- **원인**: CSS specificity에서 MDB 규칙이 우선 적용됨. Pretendard 정적(static) 폰트는 550, 650, 750 같은 중간 weight 미지원
- **해결**: `body h1 { font-weight: 750 !important; }` 형태로 override + Pretendard Variable 폰트로 교체

## 6. 테이블 스타일 개선
- **현상**: 다크모드에서 테이블이 테이블처럼 보이지 않음 — 보더가 안 보이고, 헤더 배경이 너무 밝음
- **원인**: 다크 배경 위에 보더 색상도 어두워 구분 안 됨. MDB/distill CSS가 `!important`로 보더 덮어씀
- **해결**: th 배경을 `rgba(0,0,0,0.2)`로, 보더를 `rgba(255,255,255,0.2) !important`로 설정. `table-borderless` 테이블에도 동일 스타일 적용

## 11. Sveltia CMS 로그인 Netlify Not Found 지속
- **현상**: `/admin/` 에서 로그인 시 `api.netlify.com` 으로 리다이렉트되며 "Not Found"
- **원인**: Sveltia CMS가 `backend: github`만으로는 자체 인증하지 않고 Netlify auth로 보냄. 별도 인증 프록시 설정 필요
- **해결**: `admin/config.yml`에 `base_url: https://sveltia-cms-auth.netlify.app` 추가. GitHub OAuth App의 callback URL을 `https://sveltia-cms-auth.netlify.app/callback`으로 변경

## 13. Sveltia CMS 인증 프록시 문제 재조사
- **현상**: `base_url: https://sveltia-cms-auth.netlify.app` 설정 후에도 여전히 `api.netlify.com`으로 리다이렉트되며 Not Found
- **문제**: `sveltia-cms-auth.netlify.app`는 공용 프록시가 아니라 직접 배포해야 하는 Cloudflare Worker 기반 서비스
- **원인**: Sveltia CMS의 GitHub 인증은 자체 callback 경로(`/admin/auth/callback/github`)를 지원
- **해결**: `base_url`을 `https://ucjung.github.io`로 변경, OAuth App callback URL을 `https://ucjung.github.io/admin/auth/callback/github`로 설정. 인증 방식 조사 결과 Token 방식이 가장 현실적 (OAuth는 별도 Cloudflare Worker 필요)
