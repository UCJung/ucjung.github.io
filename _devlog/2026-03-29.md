# 2026-03-29 개발일지

## 1. Heading(H1~H6) 폰트 웨이트 적용 안 됨
- **현상**: `_typography.scss`에 H1~H6 font-weight를 계층적으로 설정했으나 화면에 반영되지 않음. 개발자 도구로 확인하니 모든 heading이 `font-weight: 300`으로 표시됨
- **문제**: MDB(Material Design Bootstrap)의 CSS가 `h1, h2, h3, h4, h5, h6 { font-weight: 300 }` 으로 강제 덮어쓰고 있었음
- **원인**: CSS specificity에서 MDB 규칙이 우선 적용됨. 또한 Pretendard 정적(static) 폰트는 550, 650, 750 같은 중간 weight를 지원하지 않음
- **해결**:
  1. `body h1 { font-weight: 750 !important; }` 형태로 specificity + !important로 MDB 덮어씀
  2. Pretendard CDN을 정적 폰트 → 가변 폰트(Variable)로 변경하여 중간 weight 정확 렌더링

## 2. Sveltia CMS 로그인 시 "Not Found"
- **현상**: `/admin/` 페이지에서 "Login with GitHub" 클릭 시 `api.netlify.com` 팝업이 뜨며 "Not Found"
- **문제**: `admin/config.yml`에 `auth_type: pkce`, `app_id` 설정이 남아있어 Sveltia CMS가 여전히 Netlify 경유 인증을 시도
- **원인**: Sveltia CMS는 별도 설정 없이 `backend: github`만 있으면 자체 인증 처리. 불필요한 설정이 충돌
- **해결**: `auth_type: pkce`와 `app_id` 제거

- **원인**: Al-folio 전환 시 giscus 설정값을 채우지 않았음
